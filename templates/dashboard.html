🔎 SearchandResearch

Paste a URL, scrape it with a real headless browser, then download the results.

Scrape a URL

Crawl depth 0 = just this page. Depth 1 follows same-domain links one hop, etc. Results save into a timestamped run folder containing each page's page.html, data.json, optional screenshot.png, and a standalone report.html you can double-click to study offline. The ZIP preserves every folder and subfolder.

How to set this up & use it (click a step to expand)

1 · Install dependencies
Install Python packages, then download the browser Playwright drives: 
pip install -r requirements.txt
playwright install chromium
2 · Launch the dashboard
Run one command — it starts the local engine and opens this dashboard in your browser automatically: 
python app.py
(A plain .html file opened from disk can't drive a browser or write files, so this tiny local server does that part.)
3 · Scrape a page
Paste a URL in the box above and press Scrape. Tick Save screenshot for a full-page PNG, set a Crawl depth to follow same-domain links, and leave Respect robots.txt on to skip disallowed pages. Each scraped page produces page.html, data.json (title, links, visible text, custom fields), and optionally screenshot.png.
4 · Choose where results go
Pick a destination under Where to save:
  • Repo collections/ folder — saved inside this repository so you can review and git commit / git push it yourself (nothing is pushed automatically).
  • A local folder I choose — type any path (e.g. ~/research/site) and results are written there for you to study.
Either way each run lands in a timestamped folder. Click Download results (ZIP) for a copy of that run — every folder and subfolder included.
5 · Study the scraped content
Every run writes a standalone report.html — double-click it to browse each page's title, visible text, links, custom fields, and screenshot offline. The raw page.html and data.json sit alongside it for digging into the code.
6 · Use it from the command line
The same engine works as a CLI without the dashboard: 
python scraper.py https://example.com --screenshot
python scraper.py https://example.com --crawl-depth 1 --respect-robots
python scraper.py --urls-file urls.txt --output-dir output
7 · Extract custom fields (CSS selectors)
Create a YAML/JSON config mapping output keys to CSS selectors and pass it with --config. Values land under "fields" in each data.json: 
fields:
  page_heading: "h1"
  story_titles:
    selector: ".titleline > a"
    multiple: true
    attr: text
See fields.example.yaml in the repo.
